120  // String creation functions. Most of the string creation functions take
121  // a Heap::PretenureFlag argument to optionally request that they be
122  // allocated in the old generation. The pretenure flag defaults to
123  // DONT_TENURE.
124  //
125  // Creates a new String object. There are two String encodings: one-byte and
126  // two-byte. One should choose between the three string factory functions
127  // based on the encoding of the string buffer that the string is
128  // initialized from.
129  // - ...FromOneByte initializes the string from a buffer that is Latin1
130  // encoded (it does not check that the buffer is Latin1 encoded) and
131  // the result will be Latin1 encoded.
132  // - ...FromUtf8 initializes the string from a buffer that is UTF-8
133  // encoded. If the characters are all ASCII characters, the result
134  // will be Latin1 encoded, otherwise it will converted to two-byte.
135  // - ...FromTwoByte initializes the string from a buffer that is two-byte
136  // encoded. If the characters are all Latin1 characters, the result
137  // will be converted to Latin1, otherwise it will be left as two-byte.
138  //
139  // One-byte strings are pretenured when used as keys in the SourceCodeCache.
140  MUST_USE_RESULT MaybeHandle<String> NewStringFromOneByte(
141  Vector<const uint8_t> str, PretenureFlag pretenure = NOT_TENURED);
