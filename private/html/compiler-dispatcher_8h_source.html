37 // The CompilerDispatcher uses a combination of idle tasks and background tasks
38 // to parse and compile lazily parsed functions.
39 //
40 // As both parsing and compilation currently requires a preparation and
41 // finalization step that happens on the main thread, every task has to be
42 // advanced during idle time first. Depending on the properties of the task, it
43 // can then be parsed or compiled on either background threads, or during idle
44 // time. Last, it has to be finalized during idle time again.
45 //
46 // CompilerDispatcher::jobs_ maintains the list of all CompilerDispatcherJobs
47 // the CompilerDispatcher knows about.
48 //
49 // CompilerDispatcher::pending_background_jobs_ contains the set of
50 // CompilerDispatcherJobs that can be processed on a background thread.
51 //
52 // CompilerDispatcher::running_background_jobs_ contains the set of
53 // CompilerDispatcherJobs that are currently being processed on a background
54 // thread.
55 //
56 // CompilerDispatcher::DoIdleWork tries to advance as many jobs out of jobs_ as
57 // possible during idle time. If a job can't be advanced, but is suitable for
58 // background processing, it fires off background threads.
59 //
60 // CompilerDispatcher::DoBackgroundWork advances one of the pending jobs, and
61 // then spins of another idle task to potentially do the final step on the main
62 // thread.
