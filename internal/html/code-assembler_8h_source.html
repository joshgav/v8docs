126 // A "public" interface used by components outside of compiler directory to
127 // create code objects with TurboFan's backend. This class is mostly a thin shim
128 // around the RawMachineAssembler, and its primary job is to ensure that the
129 // innards of the RawMachineAssembler and other compiler implementation details
130 // don't leak outside of the the compiler directory..
131 //
132 // V8 components that need to generate low-level code using this interface
133 // should include this header--and this header only--from the compiler directory
134 // (this is actually enforced). Since all interesting data structures are
135 // forward declared, it's not possible for clients to peek inside the compiler
136 // internals.
137 //
138 // In addition to providing isolation between TurboFan and code generation
139 // clients, CodeAssembler also provides an abstraction for creating variables
140 // and enhanced Label functionality to merge variable values along paths where
141 // they have differing values, including loops.
